## 2024-05-15 - React Markdown Performance during Streaming
**Learning:** In a chat interface where messages are streaming in, the parent component re-renders frequently. If `react-markdown`'s `remarkPlugins`, `rehypePlugins`, or `components` props are defined inline within the functional component, they are recreated on every render. This causes `react-markdown` to re-parse the entire Markdown AST on every single chunk, leading to significant UI lag and high CPU usage.
**Action:** Always define `remarkPlugins`, `rehypePlugins`, and custom `components` statically outside of the component body. Wrap the Markdown renderer in `React.memo` so it only re-renders when the actual content string changes. Also, wrap the individual message bubbles in `React.memo` to prevent re-rendering historical messages when only the latest message is streaming.
